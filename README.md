## 快速开始

本项目支持两种使用方式：

### 方式一：在 TRAE / Claude Code / Codex 等agent中直接使用（推荐）

在当前对话中直接提出需求，AI 会调用对应 Skill：

| 场景 | 你说 | AI 做什么 | 产出 |
|------|------|---------|------|
| 简历优化 | "帮我优化简历" | JD分析→STAR重写→生成HTML | 简历文件 + 修改对照表 |
| 证据链审核 | "开启证据链" | 对每个简历声明进行C0-C3分级 | 证据链审核表 + 安全表述建议 |
| 面试拷问 | "面试拷问" | 5轮递进式拷问暴露薄弱点 | 拷问问题 + 风险评级 |
| 答案卡片 | "生成答案卡片" | 高风险问题的危险/及格/强回答 | 三级答案卡片 |
| 补强计划 | "补强计划" | 按半天/1天/3天/1周分桶 | 补强任务表 |
| 打招呼话术 | "生成打招呼话术" | 首次/已读不回/面试后感谢 | BOSS打招呼模板 |
| 投递跟踪 | "记录投递：XX公司" | 记录投递+设置跟进节奏 | 投递记录 + 统计面板 |
| HR审核简历 | "以HR角度审查简历" | AI痕迹识别+数据可信度审查 | 审核报告 + 修改建议 |
| 面试准备 | "帮我准备面试" | 技能盘点→故事库→问题预测 | 面试报告 + 学习资源 |
| 薪资谈判 | "帮我谈薪资" | 市场调研→谈判策略 | 谈判话术 |

### 方式二：在其他 AI 工具中使用

复制 `PROMPT-MERGED.md` 全部内容，粘贴到 DeepSeek / 千问 等网页版，正常对话即可。

**使用步骤**：
1. 打开 `PROMPT-MERGED.md`，全选复制
2. 打开 DeepSeek / 千问 等网页版
3. 粘贴到第一条消息，发送
4. 提供简历 + JD，开始对话

> **注意**：`PROMPT-MERGED.md` 仅包含简历优化、JD 分析、面试准备的核心流程，**不包含**投递跟踪、薪资谈判、面试复盘、HR 审核等独立 Skill 的完整功能，也无法调用 `generate_resume.py` 脚本和 `references/` 目录下的参考文档。如需完整功能，推荐使用方式一。

## 完整流程示例

### 第一阶段：简历优化 + 岗位匹配

**Step 1**：上传简历文件（PDF/Word/TXT）
```
我: 帮我优化简历，目标是 Java 后端开发岗位
AI: 请上传你的简历文件，并粘贴目标岗位 JD
我: [上传 PDF] + [粘贴 JD]
```

**Step 2**：AI 执行简历优化流程
- 提取简历文本
- 经历与技能挖掘（追问 3 轮）
- JD 三维度拆解
- JD 硬技能提取 & 核心优势生成
- 关键词匹配度分析
- **证据链分级**（可选：C0-C3 声明审核，确保面试不露馅）
- Fit Verdict 判定
- STAR 重写 + 量化强化 + 动词升级
- **真相边界降级**（"主导"→"参与模块"，"上线"→"完成demo"）
- 生成 HTML 简历

**产出**：
- 优化后的 HTML 简历
- 修改对照表
- 匹配度报告
- 岗位差距分析与补强建议
- 证据链审核表（如开启）
- BOSS 打招呼话术（核心优势确定后自动生成）

---

### 第二阶段：面试准备

**Step 1**：开始面试准备
```
我: 帮我准备 [公司名] 的 Java 后端面试
AI: 请提供目标公司的岗位 JD 和你的简历
```

**Step 2**：Phase 0 - 技能盘点与差距分析
- 梳理岗位核心技能要求
- 询问你的掌握程度（精通/熟悉/了解/不会）
- 生成差距报告
- 推荐学习资源（视频/书籍/项目）
- 预测面试高频问题

**Step 3**：Phase 1-3 - STAR 故事库
- 基于简历构建 STAR 故事
- 预测面试问题
- 准备反问清单
- 6 轴公司调研

**Step 4**：面试拷问 5 轮（新增）
- 第 1 轮：真相边界 — 你到底做了什么？
- 第 2 轮：技术深度 — 输入/输出/数据/指标
- 第 3 轮：JD 定向深挖 — 针对目标岗位专项拷问
- 第 4 轮：场景题 — 系统挂了怎么办？数据不对怎么排查？
- 第 5 轮：风险汇总 — 每个声明评级（✅/⚠️/🔴/❌）

**Step 5**：答案卡片 + 补强计划（新增）
- 对高风险问题生成三级答案（危险/及格/强回答）
- 按半天/1天/3天/1周分桶输出补强任务
- 每个任务标注解锁的简历声明和面试题

## 核心功能详解

### 证据链分级（Step 3.5，可选）

对简历中每个重要声明进行证据分级，确保面试深挖时不露馅。

| 级别 | 表述 | 所需证据 |
|------|------|---------|
| C0 | 了解 / 参与 | 笔记、课程材料 |
| C1 | 负责模块 | 代码、README、截图 |
| C2 | 设计 / 优化 | 实验、对比、bad case |
| C3 | 结果影响 | 指标定义、基线、上线记录 |

C3 声明不允许在没有指标定义和记录的情况下写入简历。

### 真相边界降级表

国内面试官对"主导""上线"等词非常敏感，以下降级在 Step 4 重写时严格执行：

| 不安全表述 | 安全降级表述 |
|-----------|------------|
| 主导系统建设 | 参与模块开发 / 负责某一子模块 |
| 上线生产 | 完成 demo / 内部试用 / 本地验证 |
| 提升准确率 30% | 基于样例对比观察到效果改善 |
| 精通 XX | 熟练使用 / 掌握 |

### 面试拷问 5 轮

递进式拷问，暴露简历薄弱点：
1. **真相边界** — 你到底做了什么？
2. **技术深度** — 输入/输出/数据/模型/指标
3. **JD 定向深挖** — 针对目标岗位的专项拷问
4. **场景题** — 系统挂了怎么办？数据不对怎么排查？
5. **风险汇总** — 每个声明评级 ✅/⚠️/🔴/❌

### 答案卡片

对高风险问题生成三级回答：
- ❌ **危险回答**：过度声称，面试必被拆穿
- ⚠️ **及格回答**：诚实但不够亮眼
- ✅ **强回答**：有证据有细节有反思

### 补强计划时间桶

按时间紧迫程度分桶，每个任务标注解锁的简历声明和面试题：
- **半天**：清理 README、收集截图、画架构图
- **1 天**：构建 eval set、添加结构化日志、写 demo 脚本
- **3 天**：添加评估指标、写单元测试、运行对比实验
- **1 周**：构建完整仓库、准备面试资料包

### BOSS 打招呼话术库

核心优势确定后自动提炼为打招呼用语，也可单独触发（"生成打招呼""打招呼话术""BOSS打招呼"）。

国内招聘平台打招呼话术，4 个阶段：
- **首次打招呼**：硬技能直击型 / 项目亮点型 / 简洁型
- **已读不回跟进**：提供新价值，不追问为什么不回复
- **面试后感谢**：2h 内发，补充面试中讨论的技术点
- **Offer 谈薪**：争取更好条件

### 主简历库（Master Resume）

所有经历的大仓库，按 JD 从中抽取对应版本：
- **主简历 JSON**：包含全部经历、技能、项目，不限篇幅，由用户自行维护
- **定向简历 JSON**：从主简历中抽取与 JD 最匹配的内容，控制在 1 页
- 每个 experience 条目带 `tags` 字段，方便按 JD 关键词筛选
- AI 不会自动创建主简历文件，用户生成定向简历后可选择归入主简历库

### HR 简历审核（独立 Skill）

以 HR 视角审查简历，建议切换 AI 模型确保客观评价：
- Phase 1：AI 痕迹识别（抽象概念 / 无具体工作 / 场景割裂 / 权责不符）
- Phase 2：量化数据可信度审查（🟢可信 / 🟡存疑 / 🔴不可信）
- Phase 3：直接修改并给出理由
- Phase 4：综合评分

### 投递跟踪（独立 Skill）

BOSS直聘/拉勾/猎聘投递跟踪与跟进节奏：
- 8 种投递状态（已投递→已读未回→已回复→面试邀约→面试完成→Offer→已拒绝→已暂停）
- 跟进节奏（7天未读→换话术，3天已读不回→跟进1次）
- 统计面板（回复率、需跟进项、周报）

### 面试复盘（SKILL-TOOLKIT.md）

面试后结构化反思，从每次面试中获取最大价值：
- 复盘模板：记录面试官问题、自己的回答、评分
- 回答评价：答得好/答得不好/完全没答上来
- 补强计划：按优先级输出补强任务
- 复盘数据积累：3+次复盘后分析高频薄弱点和改进趋势
- 与其他 Skill 协作：复盘发现的问题可联动简历降级、技能盘点、补强计划

### 国内薪资谈判（SKILL-SALARY.md 专章）

针对国内求职场景的薪资谈判指南：
- 国内薪资结构：13/14薪、年终奖、五险一金、试用期薪资
- 关键确认清单：缴纳基数、公积金比例、试用期时长等
- 应届生谈薪策略：签字费、试用期缩短、五险一金基数
- 中文反报价话术：社招版+微信/BOSS消息版
- 国内薪资计算器：到手月薪和年总包估算

| `generate_resume.py map <json> [output]` | 字段映射：将非标准字段名映射为标准模板字段 |
| `generate_resume.py version save <json> <name>` | 保存当前简历版本 |
| `generate_resume.py version list` | 列出所有已保存版本 |
| `generate_resume.py version diff <v1> <v2>` | 对比两个版本的差异 |
| `generate_resume.py version restore <name> [output_path]` | 回滚到指定版本（可选指定输出路径） |

**版本管理可选参数**：

| 参数 | 用法 | 说明 |
|------|------|------|
| `--versions-dir` | `generate_resume.py version --versions-dir <path> <子命令>` | 自定义版本存储目录（默认为脚本同目录下的 `versions/`） |

**字段映射**：`build` 命令会自动将"姓名"→name、"手机"→phone 等非标准字段名映射为标准字段，用户 JSON 字段名不匹配时无需手动修改。

## 常见问题

**Q：PDF 简历提取失败怎么办？**
A：手动复制简历文本粘贴给 AI，脚本会自动降级处理。

**Q：简历只有一页塞不下怎么办？**
A：按优先级精简内容（删除最不相关的项目或 bullet），不要扩到第二页。

**Q：没有项目经历怎么办？**
A：在经历挖掘阶段（Step 1.3）尽可能回忆课程项目、自学项目、开源贡献等。

**Q：JD 和简历不匹配怎么办？**
A：按 Fit Verdict 判定结果处理，差距大的优先补强，短期内无法弥补的调整投递策略。

**Q：证据链审核一定要开启吗？**
A：不强制。但如果你担心面试被深挖露馅，建议开启，它会帮你识别哪些声明证据不足，给出安全替代表述。

**Q：真相边界降级表是什么？**
A：国内面试官对"主导""上线""精通"等词非常敏感，应届生最容易被追问到暴露。降级表把这些不安全表述替换为安全表述（如"主导"→"参与模块开发"），确保简历内容面试时能自圆其说。

**Q：面试拷问 5 轮和面试准备有什么区别？**
A：面试准备（Phase 0-3）侧重构建 STAR 故事和预测问题，是正向准备。面试拷问是反向压力测试，模拟面试官深挖你简历中的薄弱点，帮你提前发现风险。

**Q：BOSS 上投了 20 家没回复怎么办？**
A：使用投递跟踪 Skill 记录每条投递，系统会自动提醒你哪些需要跟进（7天未读换话术，3天已读不回跟进1次），并生成对应的跟进话术。

**Q：HR 审核为什么要换 AI 模型？**
A：同一个 AI 模型优化出来的简历，让它自己审核容易"自己宽待自己"，切换模型能更客观地发现问题。

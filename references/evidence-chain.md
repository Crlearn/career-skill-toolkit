# 优势证据链方法论

> 核心理念：求职不是美化经历，而是**翻译经历**。很多人不是没能力，是讲不清能力。

---

## 一、证据链四步模型

每个优势必须走完这条链路：

```
证据 → 行为 → 能力 → 岗位信号
```

### 1.1 四要素定义

| 要素 | 含义 | 示例 |
|------|------|------|
| **证据** | 你真实做过的事情 | 做过校园 AI 工具项目 |
| **行为** | 你在证据中具体做了什么 | 访谈用户、整理需求、测试功能 |
| **能力** | 从行为中提炼出的可迁移能力 | 需求理解、沟通、文档、快速学习 |
| **岗位信号** | 这些能力对应哪些岗位的需求 | AI 应用实习 / 产品运营助理 |

### 1.2 关键原则

**没有证据的优势，只是自我感觉。**

- 每个声称的能力都必须追溯到至少一个具体证据
- 证据必须是真实的经历，不能编造
- 行为必须可描述（谁、做了什么、用什么方法、什么结果）
- 能力必须是可迁移的（不依赖特定场景）
- 岗位信号必须有 JD 或市场数据支撑

### 1.3 证据链构建流程

**Step 1：穷举证据**
- 列出所有经历（实习、项目、校园、课程、志愿、自学）
- 每个经历用 1-2 句话描述做了什么
- 不筛选，先全部列出

**Step 2：提取行为**
- 对每个证据，列出你具体做了什么动作
- 用动词开头：访谈了 X 人、整理了 Y 份文档、测试了 Z 个功能
- 关注"你做了什么"，而非"团队做了什么"

**Step 3：归纳能力**
- 将相似行为归类为能力维度
- 每个能力至少有 2 个不同证据支撑
- 区分"核心能力"（多证据支撑）和"边缘能力"（单证据）

**Step 4：映射岗位信号**
- 每个核心能力对应 2-3 个目标岗位
- 用 JD 关键词验证匹配度
- 标注匹配证据和差距

## 二、证据链模板

```markdown
## 证据链：[能力名称]

### 证据
1. [经历1]：[具体描述]
2. [经历2]：[具体描述]

### 行为
- [动词] + [对象] + [方法/工具] + [结果]
- 例：访谈了 15 位同学（对象），用半结构化问卷（方法），整理出 8 个核心需求（结果）

### 能力
- **[能力1]**：[定义]
- **[能力2]**：[定义]

### 岗位信号
| 岗位 | 匹配证据 | 差距 |
|------|---------|------|
| AI 应用实习 | 需求理解 + 用户访谈 | 缺少部署经验 |
| 产品运营助理 | 文档整理 + 数据分析 | 缺少运营工具经验 |
```

## 三、常见错误

| 错误 | 正确做法 |
|------|---------|
| "我沟通能力强"（无证据） | "在 XX 项目中访谈了 15 位用户，整理出需求文档" |
| "我学东西快"（无证据） | "2 周内自学 Python，独立完成 XX 项目的数据处理模块" |
| "我负责了 XX 项目"（只有标签） | "在 XX 项目中主导了需求分析、用户测试和文档撰写" |
| "参与了 XX"（太模糊） | "在 XX 中负责 [具体部分]，使用 [工具]，产出 [结果]" |
| 把团队成果当成个人能力 | 明确区分"我做的"和"团队做的"，只 claim 自己的贡献 |

## 四、从证据链到简历内容

证据链构建完成后，按以下规则转化为简历 bullet：

1. **每个 bullet = 一个证据 + 行为 + 结果**
2. 用强动词开头（见 `references/writing-rules.md`）
3. 包含量化数据（见 `references/writing-rules.md`）
4. 自然融入 JD 关键词（见 `references/star-methodology.md`）

**转化示例**：

```
证据链：
  证据：校园 AI 工具项目
  行为：访谈 15 位用户、整理 8 个核心需求、测试 3 个版本
  能力：需求理解、用户研究、快速迭代
  岗位信号：AI 应用实习

转化为简历 bullet：
  "访谈 15 位校园用户，整理 8 个核心需求，推动产品从 V1 迭代至 V3，
   用户满意度从 60% 提升至 85%"
```

## 五、从证据链到面试故事

每个核心能力准备一个完整的 STAR 故事（见 `SKILL-INTERVIEW.md`），结构：

```
S：[证据的背景]
T：[你在其中的任务]
A：[你的具体行为 — 来自证据链的"行为"列]
R：[量化结果]
Reflection：[从这段经历中学到的能力 — 来自证据链的"能力"列]
```

**关键**：面试追问时，所有回答都必须能追溯到证据链中的具体行为。没有证据支撑的回答会显得空洞。

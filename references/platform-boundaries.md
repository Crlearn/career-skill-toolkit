# 平台边界与数据使用规范

> 明确对不同求职平台的数据使用边界，确保合法合规使用。

---

## 一、核心原则

本项目的所有功能**只在用户明确授权且用户主动提供的信息范围内工作**。任何对外部平台的自动化操作，都必须由用户手动确认和执行。

---

## 二、允许的数据来源

| 来源 | 允许的使用方式 | 说明 |
|------|--------------|------|
| **用户粘贴** | 用户从平台复制 JD 文本粘贴给 AI | 最推荐的方式，无合规风险 |
| **上传截图** | 用户上传平台页面的截图（PNG/JPG） | AI 通过 OCR 或视觉识别分析 |
| **导出文件** | 用户从平台导出的 CSV/Excel 文件 | 仅分析用户主动提供的文件 |
| **浏览器可见页面** | 用户打开页面后让 AI 读取可见内容 | 不绕过登录验证，不爬取隐藏内容 |
| **岗位链接** | 用户提供公开岗位链接 | 仅获取该链接的公开可见内容 |
| **手动录入** | 用户手动输入岗位信息 | 完全由用户控制 |

---

## 三、不允许的操作

| 操作 | 原因 |
|------|------|
| **批量爬取招聘平台** | 违反平台服务条款，可能涉及不正当竞争 |
| **绕过登录验证/验证码/风控** | 违反平台安全机制，可能涉及非法获取计算机信息系统数据 |
| **自动投递简历** | 替代用户决策，且可能被平台识别为异常行为 |
| **自动私信 HR/招聘方** | 替代用户沟通，可能被平台封禁 |
| **存储用户账号凭据/Cookie** | 严重安全隐患，凭据泄露可能导致账号被盗 |
| **承诺面试/Offer/薪资** | 虚假承诺，AI 无法控制招聘方决策 |
| **用于企业侧候选人排名/录用决策** | 超出求职辅助范围，可能涉及就业歧视 |

## 四、平台特定说明

### BOSS 直聘

| 项目 | 说明 |
|------|------|
| 可处理 | 用户复制的 JD 文本、岗位截图、导出表、浏览器可见页面 |
| 可生成 | 打招呼话术、投递准备清单、跟进提醒（用户手动执行） |
| 不可做 | 自动发送消息、自动刷新简历、自动批量投递、绕过"每日沟通上限" |

### 拉勾

| 项目 | 说明 |
|------|------|
| 可处理 | 用户复制的 JD 文本、岗位截图、导出表 |
| 可生成 | 投递策略建议、匹配度分析 |
| 不可做 | 自动投递、自动刷新、批量操作 |

### 猎聘

| 项目 | 说明 |
|------|------|
| 可处理 | 用户复制的 JD 文本、岗位截图、导出表 |
| 可生成 | 简历定向优化建议、打招呼话术 |
| 不可做 | 自动投递、自动私信猎头 |

### 领英（LinkedIn）

| 项目 | 说明 |
|------|------|
| 可处理 | 用户复制的 JD 文本、公开可见页面 |
| 可生成 | 英文简历措辞建议、申请策略 |
| 不可做 | 自动申请 Easy Apply、自动发送 Connection 请求 |

---

## 五、推荐措辞

在描述本项目功能时，建议使用以下措辞：

**推荐使用**：
- "监督式精准投递助手"
- "岗位整理与匹配分析"
- "生成投递准备清单"
- "用户确认后手动投递"
- "投递策略建议"

**避免使用**：
- "全自动海投"
- "自动刷岗位"
- "保证面试"
- "绕过平台限制"
- "代投简历"

---

## 六、来源记录要求

分析每个岗位时，必须记录以下信息：

```
platform:      BOSS直聘 / 拉勾 / 猎聘 / 领英 / 其他
company:       公司名称
title:         岗位名称
url:           岗位链接（如有）
observed_at:   观察日期（YYYY-MM-DD）
source_type:   user_paste / screenshot / csv_export / browser_visible / manual_entry
```

---

## 七、合规声明

本项目是一个**本地求职辅助工具**，不提供：
- 自动化简历投递服务
- 招聘平台数据采集服务
- 面试/Offer 保证
- 候选人排名或录用决策支持

用户在使用本项目的任何功能时，应自行遵守各平台的服务条款和相关法律法规。

# 面试拷问与答题卡

> 核心理念：在真实面试前，先用拷问式问题暴露简历中的薄弱声明，提前准备。

---

## 一、面试拷问 5 轮框架

### Round 1：真实边界
验证用户实际做了什么。
```
"你说主导了 XX 项目，具体哪些模块是你写的？哪些是别人写的？"
"你说优化了效果，优化前 baseline 是多少？"
```

### Round 2：技术深度
验证 input/output、数据、模型、系统、指标。
```
"你的 RAG 系统 chunk 策略是什么？embedding 模型用的什么？top_k 多少？"
"你的 rerank 模型输入输出是什么格式？"
```

### Round 3：JD 专项深挖
针对目标岗位的核心要求提问。
```
"JD 要求部署经验，你的模型部署在什么环境？用什么框架？延迟多少？"
"JD 要求多轮对话，你的 session 管理怎么做的？"
```

### Round 4：场景题
失败、延迟、质量下降、数据漂移、权限等异常场景。
```
"如果你的检索结果大量幻觉，你怎么排查？"
"如果模型推理延迟从 200ms 涨到 2s，你怎么定位？"
"如果用户反馈排序变差了，你怎么分析？"
```

### Round 5：风险总结
对每个声明给出判定：
- **可回答**：有充分证据和准备
- **需补证据**：有部分准备但不充分
- **高风险**：简历声明无法在面试中自圆其说
- **不能写**：完全没有证据支撑

### 提问原则

**差**：
```
请介绍一下 RAG。
```

**好**：
```
你说优化了 RAG 召回，具体是 chunk、embedding、top_k、rerank 还是 prompt 改了？
每一步怎么证明有效？
```

每个问题必须包含：为什么重要 + 什么证据能回答。

## 二、答题卡 3 级模板

对每个高风险面试问题，准备三个级别的回答：

### 级别定义

| 级别 | 特征 |
|------|------|
| **危险回答** | 夸大、隐藏缺口、用模糊词汇 |
| **可接受回答** | 承认边界、解释真实工作 |
| **强回答** | 补充证据、权衡取舍、失败案例、下一步计划 |

### 答题卡格式

```
问题: [面试官可能问的问题]
为什么有风险: [简历中哪个声明会触发这个问题]

危险回答:
[夸大的回答 — 绝对不要用]

可接受回答:
[承认边界的回答 — 基本安全]

强回答:
[有证据支撑的回答 — 面试加分]

需要的证据: [要达到"强回答"需要什么]
简历措辞降级: [如果无法达到"强回答"，简历中应该怎么改写]
```

### 示例

**问题**：你说提升了搜索排序效果，指标是什么？

**危险回答**（绝对不要用）：
```
感觉排序结果更好了，用户体验提升很多。
```

**可接受回答**（基本安全）：
```
目前没有线上指标，所以我不会写提升比例。我做的是长尾 query bad case 分类
和 rerank baseline 复现。
```

**强回答**（面试加分）：
```
我用固定 query-doc 样例集对比 BM25、embedding 召回和 cross-encoder rerank
的结果，记录 NDCG@10 与 bad case 类型。现阶段只把它写成离线分析和
baseline 复现，不写线上提升。
```

**需要的证据**：query-doc 样例集、NDCG 计算脚本、对比结果表格
**简历措辞降级**：如果无法提供上述证据，简历改为"分析长尾 query 相关性失败样例，复现 cross-encoder rerank baseline"

### 规则

- 如果用户缺乏证据，"强回答"可以是"我目前不会在简历中写这个声明"
- 每个答题卡必须关联到简历中的具体声明
- 答题卡应覆盖所有"高风险"和"需补证据"的声明
